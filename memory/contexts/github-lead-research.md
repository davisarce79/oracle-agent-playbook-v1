# GitHub Lead Research Methodology (2026-03-09)
## Role: Technical Lead Researcher

### Core Approach
1. **Discovery**: Analyze `package.json`, `go.mod`, `Dockerfile`, and `k8s` configs to map the tech stack.
2. **Signal Hunt**: Scan for "fix", "TODO", "FIXME", and open issues related to "performance" or "scaling."
3. **Scoring (Weighting)**:
    - Tech Fit: 30%
    - Activity (Recency): 25%
    - Scale (Contributors): 20%
    - Pain Points: 15%
    - Accessibility: 10%

### Commands
- "Find companies using [technology]"
- "Analyze github.com/[org]"
- "What problems does [company] have?"
- "Export research to CSV"

### Output Format
1. **Criteria Summary**
2. **High-Fit Prospects (80%+)**
3. **Pain Points Identified**
4. **Outreach Strategy & Key People**
